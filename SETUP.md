## 🚀 Development Workflow

### Prerequisites
- PHP 8.3+ (with extensions: mbstring, pdo, mysql, openssl, tokenizer, xml)
- MySQL 8.0+
- Node.js 18+ (includes npm)
- Composer

### Setup Environment

1. **Install PHP dependencies:**
   ```bash
   composer install
   ```

2. **Install Node dependencies:**
   ```bash
   npm install
   ```

3. **Configure environment:**
   ```bash
   # Already done, but if needed:
   cp .env.example .env
   php artisan key:generate
   ```

4. **Create database:**
   ```bash
   # Create MySQL database named 'yoklama'
   mysql -u root -e "CREATE DATABASE yoklama;"
   ```

5. **Run migrations:**
   ```bash
   php artisan migrate
   ```

### Development Server

**Terminal 1 - Run Laravel server:**
```bash
php artisan serve
# App available at: http://localhost:8000
```

**Terminal 2 - Run Vite dev server:**
```bash
npm run dev
# Hot module reloading enabled
```

## 🎨 Frontend Architecture

### Layout System

The application uses **Authenticated** and **Guest** layouts:

**AuthenticatedLayout** (`resources/js/Layouts/AuthenticatedLayout.jsx`):
- **Fixed Top Navbar**: Logo, app name, user dropdown, responsive menu toggle
- **Fixed Left Sidebar**: Navigation links (expandable/collapsible)
- **Main Content Area**: Responsive to sidebar state
- **Mobile Optimized**: Sidebar toggles on mobile, overlay backdrop

**Features:**
- Responsive hamburger menu (hidden on desktop, visible on mobile)
- User profile dropdown with Profile & Logout options
- Dynamically generated navigation items
- Smooth transitions with Tailwind CSS

### Component Structure

**Reusable Components** (`resources/js/Components/`):
- **Form Components**: `TextInput.jsx`, `Checkbox.jsx`, `InputLabel.jsx`, `InputError.jsx`
- **Button Components**: `PrimaryButton.jsx`, `SecondaryButton.jsx`, `DangerButton.jsx`
- **UI Components**: `Dropdown.jsx`, `Modal.jsx`, `NavLink.jsx`
- **Layout Components**: `ApplicationLogo.jsx`

**Page Components** (`resources/js/Pages/`):
- Represent individual routes
- Use layouts via `AuthenticatedLayout` or `GuestLayout`
- Use props passed from Laravel controllers via Inertia

### Routing

Routes are defined in `routes/web.php`:
```php
Route::get('/', function () {
    return Inertia::render('Welcome');
});

Route::middleware('auth')->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index'])->name('dashboard');
    Route::get('/profile', [ProfileController::class, 'edit'])->name('profile.edit');
    // ... more authenticated routes
});

// Auth routes via Breeze (register, login, etc.)
require __DIR__ . '/auth.php';
```

## 🔐 Authentication Flow

1. **User Registration/Login** → Breeze pages (`Resources/js/Pages/Auth/`)
2. **Session Created** → Stored in database (`SESSION_DRIVER=database`)
3. **Middleware** → `auth` middleware protects authenticated routes
4. **User Props** → `usePage().props.auth.user` available in React components
5. **Logout** → `POST /logout` via Breeze

## ⚙️ Troubleshooting

### Issue: Styles not loading
**Solution**: Ensure `npm run dev` is running and check browser console for build errors.

### Issue: 419 CSRF Token Mismatch
**Solution**: Inertia handles CSRF automatically. Ensure session is properly configured in `.env`.

### Issue: React components not rendering
**Solution**: Check that page component is in `resources/js/Pages/` and route returns `Inertia::render('PageName')`.

### Issue: Path alias (@/) not working
**Solution**: Ensure `jsconfig.json` is configured correctly and Vite is restarted.
